# GEMINI.md

## Project Overview

Toodl API is the backend service for a todo management application. It provides a RESTful interface for user authentication, list management, and task scheduling. The application features email-based reminders and is being updated to support browser push notifications.

### Main Technologies

- **Runtime:** Node.js 24 (Alpine)
- **Package Manager:** pnpm 10+ (via Corepack)
- **Language:** TypeScript (configured for ESM and NodeNext)
- **Framework:** Express.js with [TSOA](https://tsoa-community.github.io/docs/) for automatic OpenAPI/Swagger generation and route management.
- **Database:** PostgreSQL with [Prisma ORM](https://www.prisma.io/).
- **Testing:** [Vitest](https://vitest.dev/) for unit and integration tests.
- **Background Tasks:** [node-cron](https://github.com/node-cron/node-cron) for scheduled email notifications.
- **Infrastructure:** Docker/Docker Compose for consistent development and production environments.

## Architecture

The project follows a **Controller-Service** pattern:

- **Controllers (`src/controllers/`):** TSOA-decorated classes that handle HTTP requests, validate input, and define OpenAPI metadata.
- **Services (`src/services/`):** Encapsulate business logic and interact with the Prisma client for database operations.
- **Routes:** Automatically generated by TSOA in `src/routes.ts` based on controller decorators.
- **Authentication:** Session-based authentication using `express-session` and `session-file-store`. Security is handled via TSOA `@Security` decorators.

## Building and Running

The following commands are defined in `package.json`:

- **Development:**

  ```bash
  pnpm dev
  ```

  Starts the server with `tsx --watch` for hot-reloading. Also triggers TSOA code generation.

- **Build:**

  ```bash
  pnpm build
  ```

  Generates TSOA routes/specs and compiles TypeScript to the `dist/` directory.

- **Production:**

  ```bash
  pnpm serve
  ```

  Runs Prisma migrations and starts the compiled server from `dist/server.js`.

- **Testing:**

  ```bash
  pnpm test
  ```

  Runs the Vitest test suite and generates a coverage report.

- **Linting & Formatting:**
  ```bash
  pnpm lint          # Run ESLint
  pnpm format:fix    # Run Prettier fix
  ```

## Development Conventions

### ESM & Imports

The project uses ECMAScript Modules (`type: module`).

- **Requirement:** All relative imports must include the `.js` file extension (even if importing `.ts` files), e.g., `import { MyService } from "./MyService.js"`.
- **Subpath Imports:** The project uses the `#/*` alias (defined in `package.json`) to refer to the `src/` directory, e.g., `import { prisma } from "#/prisma.js"`.

### Database Changes

1. Modify `prisma/schema.prisma`.
2. Generate a migration: `pnpm dlx prisma migrate dev --name <description>`.
3. Update the Prisma client: `pnpm dlx prisma generate`.

### Routing & Documentation

1. Add or modify controllers in `src/controllers/`.
2. TSOA will automatically update `src/routes.ts` and `src/swagger.json` during build or development.
3. Access Swagger UI at `/docs`.

### Testing

- Place test files in the `test/` directory.
- Use Vitest's `it` or `test` blocks.
- Ensure new features or bug fixes are covered by relevant tests.
